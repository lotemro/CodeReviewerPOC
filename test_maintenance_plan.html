Test Maintenance & Cleanup Plan

This document outlines the proposed final test suite for the repository. The goal is to maintain a professional, lean codebase with high coverage while removing redundant files.

Test File Category Status Reasoning
tests/test_config.py Unit (Core) MAINTAIN Essential for verifying environment variable loading and defaults.
tests/test_repository.py Unit (Data) MAINTAIN Verifies database operations, state transitions, and TTL logic in isolation.
tests/test_llm_client.py Unit (LLM) MAINTAIN Verifies HTTP communication with the Ollama API using mocks.
tests/test_engine.py Unit (Engine) MAINTAIN Verifies the core analysis logic and prompt construction.
tests/test_orchestrator.py Unit (Orchestration) MAINTAIN Verifies business workflows (caching, concurrency) without the HTTP layer.
tests/test_api.py Unit (API) MERGE & REMOVE Overlap with test_flow.py. Unique validation cases (invalid extension, 404) will be moved to test_flow.py.
tests/test_flow.py Integration / E2E MAINTAIN The primary system-wide test. Will be expanded to include all API edge cases.
tests/bulk_runner.py Manual Tool MAINTAIN Extremely useful for live verification and stress testing with the 20 sample files.
tests/parallelism_test.py Integration Tool MAINTAIN Explicitly requested to prove strict concurrency enforcement in a live environment.
tests/sample_files/ Test Data MAINTAIN Necessary for the bulk_runner.py utility.

Summary of Changes

  • Consolidation: test_api.py will be deleted after its logic is merged into test_flow.py, creating a single robust entry point for API testing.
  • Independence: Unit tests for each internal module (db, engine, llm, config) remain separate to ensure fast feedback during development.
  • Observability: Tools like bulk_runner are preserved to provide a "live" dashboard of system performance.

Awaiting approval to proceed with the merge and cleanup.
